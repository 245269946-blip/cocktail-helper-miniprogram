🔍 搜索页 pages/search/

智能建议构建
search.js

buildSuggestionGroups(keyword) 核心逻辑：

  1. key 为空 → 展示热门/按酒底/按材料
  2. key 命中 base name → 展示该基酒的精准匹配/条件搜索/低门槛方向
  3. key 命中 ingredient → 展示材料搭配/条件搜索/goodWith
  4. key 命中 searchMapping → 展示精准匹配/条件搜索/继续缩小
  5. key<2 或无匹配 → 热门兜底
  6. 有关键词但无匹配 → 生成条件搜索建议

Hero卡 + 分区扩展
results.js (12.9KB - 最复杂页面逻辑)

三种模式：

  • mode=search: recommend.search() → 多重兜底(别名→别名目标→中文名匹配→全字段模糊) → buildDecision() 构建分区
  • mode=tag: filterByTag → buildGeneralDecision() 按意图排列分区
  • mode=ids: getItemsByIds → 同上

buildDecision() 支持按 base/ingredient/通用 三种决策逻辑，动态排序分区池(easier/convenience/sweeter/lower/fresh/party/batch 等)

results.wxml

结构：导航栏 → 结果统计头部(decoImage装饰) → 关键词栏 → Hero Card(badgeText/nameEn/tags/quote/metaTags/CTA + illustration) → 更多推荐(ext-card列表) → 大家也在搜(trending横滑) → 空状态
