## 0.1 产品本质定义

后续所有视觉资产、image2 生图、小程序页面氛围，
必须始终以：

**「layer1 / layer2 / layer3 原始参考图」**

作为最高视觉锚点。

### 这不是：

❌ 一个传统调酒工具
❌ 一个专业酒谱产品
❌ 一个夜店风产品
❌ 一个产品感很重的互联网 UI
❌ 一个"AI 生成感"很强的项目

### 而是：

✅ 一本真实存在的、
**「亚洲城市夜晚微醺生活杂志」**
（气质参考：日本便利店的安静时刻 × 周末晚上一个人的房间 × 夏天晚风里的阳台）

用户进入后感受到的，
不是：
> "我在使用一个工具"。

而是：
> "今晚下班后，
> 好像有人递给我一本：
> 关于微醺、夜晚、一个人生活、
> 便利店、夏夜空气的小杂志。
> 
> **翻开第一页，就想倒一杯。"**

## 0.2 核心情绪关键词（所有图必须围绕）

以下是每张图的"情绪检查表"——出图前、验收时逐条核对：

| 关键词 | 情绪指向 | 出图时的具体表达 |
|--------|---------|----------------|
| 微醺 | 放松但不醉 | 饮品呈现方式：半满杯、自然状态、不刻意摆拍 |
| 晚风 | 时间感 | 光线偏傍晚/入夜暖光或冷色窗光 |
| 便利店冷柜蓝光 | 场景标识色 | 蓝白冷光元素出现在关键氛围图中 |
| 风铃 | 日式/夏天听觉联想 | 装饰性元素，轻盈、有动态暗示 |
| 小票 | 生活痕迹 | 手写纸条/收银条/便利贴——有人用过的感觉 |
| 夏夜 | 季节温度 | 整体色调偏暖但不过热，有夜晚凉意 |
| 安静 | 不吵不闹 | 构图简洁、无多余元素、大量呼吸空间 |
| 放松 | 无压力 | 不专业、不严肃、不教程化、不清单化 |
| 一个人的小仪式感 | 私密感 | 单人视角、小桌面、独饮场景 |
| 日常里的治愈 | 温柔 | 温暖但不甜腻、安静但有温度 |
| 淡淡孤独但不 emo | 情绪分寸 | 有孤独感但不是丧，是舒服的那种一个人 |
| 深夜还亮着灯的房间 | 归属感 | 窗光/台灯/室内暖光源 |
| 日杂感 | 画风总控 | 日本生活杂志排版气质、纤细、克制 |
| 生活感 | 真实度 | 不是棚拍感，是有生活痕迹的真实场景 |
| 呼吸感 | 留白与密度 | 画面填充率 ≤ 65%，必须有"空"的区域 |
| "想截图保存" | 最终检验标准 | 如果用户不想截图分享，这张图就不及格 |

## 0.3 所有图的核心目标

**不是：**
> "信息表达"

**而是：**
> "让人产生：
> **今晚真的想喝一杯** 的情绪。"

这意味着：
- 图的首要任务是 **唤起情绪**，不是传递信息
- 信息交付由前端文字/CSS 承担，图片只负责 **氛围感染力**
- 如果一张图"信息完整但毫无情绪"——不及格
- 如果一张图"信息模糊但让人想喝一杯"——及格以上

## 0.4 绝对红线（宁可……也不要……）

### ✅ 宁可：

| 做 | 效果 |
|----|------|
| 更克制 | 少即是多，一张图讲好一件事 |
| 更安静 | 不要每个角落都有东西 |
| 更留白 | 画面 35%+ 必须是"空" |
| 更像一本杂志内页 | 排版感、阅读感、翻阅感 |

### ❌ 也不要：

| 绝对避免 | 原因 |
|---------|------|
| Banner 感 | 太电商、太广告、太"推广位" |
| 电商感 | 像商品详情页、像淘宝/京东 |
| 小红书模板感 | 高饱和、大字标题、拼贴感 |
| AI 拼贴感 | 元素之间没有有机关系 |
| UI 过满 | 信息堆积、无处喘息 |
| 信息堆积 | 试图在一张图里塞太多内容 |
| 强商业化 | 像促销活动页、不像生活方式 |
| 高饱和网红感 | Instagram 滤镜风、太"网感" |
| **完美电商摆拍** | **像棚拍商业静物——违反 0.8 真实生活感原则** |
| **完美静物摄影感** | **影棚级三点布光、无缝白底——太假** |
| **所有元素整齐居中对称** | **像用尺子量过——真实生活不是这样的** |
| **Pinterest AI moodboard 感** | **好看但没有具体时间/空间/人——是氛围空壳** |
| **过度符号化日式元素堆叠** | **风铃+障子+榻榻米+和风扇=刻意的"日本风产品"，见 0.8.4** |
| **"为了高级而高级"的极简空镜** | **高级但空洞——需要加入生活痕迹(0.8.2)** |

## 0.5 image2 的职责边界

image2 负责的是：

> **「情绪氛围插画层」**

image2 **不负责**：

- ❌ UI 设计稿（按钮布局、页面结构）
- ❌ 完整产品界面（带导航栏、TabBar 的整屏）
- ❌ 文字排版（标题、正文、标签文字——全部由前端渲染）
- ❌ 交互组件（输入框、开关、列表项）
- ❌ 功能性图标（搜索、返回、收藏——CSS 或已有图标）

**图片只负责提供：「生活感与情绪」。UI 本身必须保持克制。**

## 0.8 ⬛ 真实生活感原则（v2.1 新增，与 0.1 同级优先）

> **这是对 v2.0 最关键的修正方向：**
> 从"高级设计项目感" → 回归到"真实存在的生活片段"。

### 0.8.1 核心定义

所有图：

**不要像：**
> "已经摆拍完成的商业静物。"

**而应该像：**
> "某个真实存在的人，
> 刚刚准备开始今晚第一杯时，
> 被顺手记录下来的生活片段。"

### 0.8.2 image2 出图时 —— 必须「允许出现」的生活痕迹

以下是 **加分项**（不是每张图全部要有，但一张图至少有 2-3 个）：

| 允许出现的痕迹 | 为什么重要 | 视觉表达 |
|---------------|-----------|---------|
| 冰块轻微融化 | 说明时间在流逝、不是刚摆拍的 | 杯中冰块大小不均匀、边缘圆滑、有融化水痕 |
| 杯壁冷凝水珠 | 真实的物理现象、冷感的视觉证据 | 玻璃杯外壁细密水珠、有自然流淌痕迹 |
| 小票折痕 / 卷边 | 有人拿过、塞进口袋过 | 纸条不是完全平整的、边缘微卷 |
| 桌面轻微水渍 / 圆形杯印 | 刚放过别的杯子、这个位置有人坐过 | 桌面有不规则浅色水渍圈 |
| 杯垫轻微歪斜 | 不是用尺子摆的 | 杯垫旋转了 5-15°、不完全对齐杯底 |
| 桌面不是绝对整洁 | 真实的家/便利店桌面就是这样的 | 远处模糊区域可有杂物暗示（书角/手机边/纸巾）|
| 光线轻微不均匀 | 室内灯光就是这样 | 一侧稍亮一侧稍暗、有自然的明暗过渡 |
| 构图不完全对称 | 手机随手拍/素描速写感 | 主体偏左或偏右一点、地平线微微倾斜也可接受 |
| 有"刚被使用过"的痕迹 | 人存在过的证据 | 吸管被咬过一端、杯沿有唇印暗示、餐巾纸揉皱 |

### 0.8.3 绝对禁止的"过度精致"

| ❌ 禁止 | 对应的真实感修正 |
|-------|-----------------|
| 完美电商摆拍（白色无缝背景、三点布光） | 用真实桌面环境光代替 |
| 完美静物摄影感（影棚级锐度、完美景深） | 允许轻微模糊、允许不完美对焦 |
| 所有元素整齐居中对称 | 故意偏移 5-10%、让构图"不那么刻意" |
| "为了高级而高级"的极简空镜 | 加入生活痕迹——哪怕只是一个杯印 |
| 太像 Pinterest AI moodboard | 加入具体的时间/空间线索——"这是一个周二的晚上11点" |
| 太像小红书"日系模板"（樱花+滤镜+统一色调） | 去掉符号化日式元素、回到真实城市夜晚 |
| 过度符号化的"日本元素堆叠"（风铃+障子+榻榻米+和风扇） | 见 0.8.4 方向修正 |

### 0.8.4 关键方向修正：「亚洲城市夜晚」>「日本风」

当前风险：
> 产品正在变成"日本风产品"。

应该变成：
> **"亚洲城市夜晚生活感"。**

**方向调整对照：**

| ❌ 减弱 | ✅ 增强 | 原因 |
|---------|--------|------|
| 强和风符号堆叠 | 夏夜空气的温度感 | 风铃可以保留一处（启动页），但不要泛滥 |
| 刻意日系排版（竖排假名感） | 自然阅读流向 | 像一本你随手翻开的书，不像"日式设计作品" |
| 统一低饱和"日系滤镜" | 真实光线下的自然色温 | 不同场景可以有不同色温——便利店偏蓝、家中偏暖 |
| "日式元素"作为装饰目的 | 元素必须有功能性/叙事性 | 风铃=夏天的声音；冷柜蓝光=便利店的记忆 |
| 抽象的"氛围感" | 具体的"某个时刻" | 不是"一个美好的晚上"，而是"周二晚上11点，刚下班" |

**新的氛围关键词优先级调整：**

```
最高优先（每张图必须有至少 2 个）：
  ① 深夜灯光          — 台灯/窗外的城市光/路灯光
  ② 冷柜蓝光          — 便利店的标志性颜色记忆
  ③ 一个人回家后的放松 — 脱鞋、换衣服后、卸下防备的状态
  ④ 城市夜晚里的安静感 — 远处车流声被关在窗外

高优先（强烈推荐）：
  ⑤ 夏夜空气        — 开窗后的那种温度
  ⑥ 使用痕迹        — 0.8.2 中列出的所有细节

保留但不滥用：
  ⑦ 风铃           — 仅限 1-2 处装饰性用法（启动页），不要成为主视觉元素
  ⑧ 日杂排版感      — 作为整体气质，不作为装饰手段
```

### 0.8.5 饮品本身必须是视觉主角

当前风险：
> 氛围已经很强，但"酒"可能变成了背景板。

**修正原则：**

用户最终的反应应该是：
> **"真的有点想喝。"**

而不是：
> **"觉得设计很好看。"**

### image2 出图时，饮品必须强化的物理质感：

| 质感维度 | 具体要求 |
|---------|---------|
| **液体通透感** | 能看到液体内部的光线折射、不同深度的颜色层次 |
| **冰块层次** | 冰块不是白色方块——能看到冰的裂纹、气泡、边缘透明度变化 |
| **杯型质感** | 玻璃杯壁的厚度、重量感、手持时的握持暗示 |
| **冷感** | 整体色温偏凉（即使暖光环境下，饮品区域也有冷调）、让人想碰 |
| **气泡** | 如果是含气饮料（汤力水/苏打/起泡酒），气泡要可见且自然上升 |
| **杯壁水珠** | 冷凝水的疏密分布要真实——底部密、上部稀 |
| **光线穿透液体** | 光从液体背后或侧后方穿过时的高光带——这是"好喝"的关键视觉信号 |
| **液面反光** | 平静液面的微弱倒影——增加真实感和诱惑力 |

**构图上：**
- 饮品主体占画面面积 ≥ 40%（氛围图）/ ≥ 60%（特写图）
- 饮品放在视觉重心或黄金分割点上
- 如果画面中有其他元素，它们的作用是 **衬托饮品的吸引力**，而不是分散注意力

> **一句话总结本章（v2.1）：**
>
> 我们做的不是一个"调酒工具"的 UI，
> 而是一本 **「会让人想在深夜打开、慢慢翻看、甚至截图保存」**
> 的 **亚洲城市夜晚微醺生活杂志**。
>
> 它不精致得像 AI 设计作品，
> 而真实得像 **某个周二晚上11点、一个人刚下班回家、
> 准备倒第一杯时、被顺手拍下来的生活片段。**
>
> **所有决策，以此为准。**

- ❌ 产品流程图 / PRD 原型

- ✅ 年轻化、微情绪、微陪伴、轻松社交
- ✅ 高颜值收藏感 —— 用户想截图分享
- ✅ 参考：tipsy / 小红书酒饮 / ins 风调酒 / 日咖夜酒 / Apple 式简洁卡片

- ✅ **核心差异**：不是"好看的工具"，而是一本能让人产生**"真的有点想喝"**情绪的生活记录
- ✅ **v2.1 关键修正**：不追求"高级设计感"，而追求**"真实生活片段感"**——见第 0 章 0.8

## 五、小程序最佳视觉结构建议

### 5.1 图片加载策略

```
策略分层：
┌─────────────────────────────────────────────┐
│  P0 图片（6张）                                  │
│  → 本地打包，预加载，不依赖网络             │
│  → 进入 app 即刻可见                              │
│  → 尺寸控制在 500KB 以内/张（WebP 更佳）        │
├─────────────────────────────────────────────┤
│  P1 图片（~30张）                                 │
│  → 按需加载（搜索到某基酒时才加载该系 Hero）   │
│  → 可做懒加载或渐进式                             │
│  → 尺寸 300-800KB/张                            │
├─────────────────────────────────────────────┤
│  P2 图片（56-86 张）                               │
│  → 按需加载，fallback 到 CSS 兜底              │
│  → 可做首屏不加载，滚动到再 load                │
│  → 尺寸 100-300KB/张                           │
└─────────────────────────────────────────────┘

总包体积估算：
  P0: 6 × 400KB ≈ 2.4 MB（必选）
  P1: 30 × 500KB ≈ 15 MB（强烈推荐）
  P2: 70 × 150KB ≈ 10.5 MB（锦上添花）
  总计：约 28 MB（完全可控）
```

### 5.2 前端应多依赖 CSS、少依赖图片的地方

| 效果 | 推荐方式 | 原因 |
|------|---------|------|
| 卡片底色/圆角/阴影 | **CSS gradient + box-shadow** | 轻量、可动态变色、不受图片限制 |
| 主题色切换（15套 palette） | **CSS 变量 + JS 动态 class** | 已实现，无需为每套色出图 |
| 酒杯/液体变化样式 | **CSS `.illo-*` 类体系** | 已实现 6 套液体变体 |
| 标签/按钮/pill | **CSS** | 轻量、易维护 |
| 便利贴/小票/对话气泡 | **CSS + clip-path** | 已实现，手绘感来自 CSS 不是贴图 |
| 装饰性分隔线/点线 | **CSS border / background** | 不需要图片 |
| 图标类（搜索/收藏/分享） | **CSS 伪元素 或 Unicode** | 不需要图片 |
| 简单的基酒/材料标识 | **3KB SVG 小图标（已有）** | 作为 fallback 就够了 |
| 粒子效果（气泡/星星） | **CSS 伪元素动画** | 比 gif 图片更灵活 |

### 5.3 图片与 CSS 的协作模式（核心架构）

```
推荐的"三层叠加"结构：

Layer 1 ─── CSS 渐变/纯色背景（底色 + 微妙氛围光斑）
  ↓  app.wxss .page 已有多重 radial-gradient
  ↓ 例：radial-gradient(circle at 14% 2%, rgba(156,225,217,0.34) 0, ...)

Layer 2 ─── AI 生成的场景插画（image component）
  ↓ mode="aspectFill" 或 mode="aspectFit"
  ↓ opacity: 0.88-0.92 让它融入而非突兀

Layer 3 ─── CSS 渐变遮罩（splash-overlay 模式）
  ↓ 从下到上：rgba(底色, 0.95) → rgba(底色, 0.3) → transparent
  ↓ 确保上方留白区能看清文字

Layer 4 ─── 前端文字/按钮/标签（绝对定位浮在上面）
  ↓ 手写体字体 + 极淡配色 + 无硬边框
```

### 5.4 字体策略（微信小程序限制下的最优解）

```
微信小程序不支持自定义字体包（除非用 wx.loadFontFace 且用户授权）
所以字体方案：

方案 A（推荐 - 用系统近似字体模拟）：
  - 标题：font-weight: 300 + font-style: italic + letter-spacing 加宽
  - 正文：-apple-system, BlinkMacSystemFont, "Helvetica Neue", sans-serif
  - 手写感来源：通过图片中的手写字（如小票文字、气泡文字画在图里）
  - 关键短文案可以用图片代替文字（如"今夜先来一杯"大标题可以做成图）

方案 B（如果允许用户授权字体）：
  - wx.loadFontFace 加载 Google Fonts
  - 推荐：Noto Serif JP（标题）+ Zen Maru Gothic（正文）+ Hi Melody（手写）
  - 注意：这会增加首次加载时间，需做 fallback

方案 C（当前 v3 预览采用的方式）：
  - HTML 预览用 Google Fonts
  - 小程序端用系统字体 + 通过图片补偿手绘感
```

### 6.2 光影规范

```

```
整体基调：
  - 底色范围：#faf8f4 ~ #f0e8d8（暖米白到淡奶茶色）
  - 饱和度：整体去饱和 20-30%（像洗旧的照片/印刷品）
  - 明度：整体偏亮（中高调），避免大面积深色

分类色盘（与 visualSystem.js 15 套 palette 对齐）：

  - 伏特加/清爽 → #3c9fc4 冷蓝（极淡）
  - 咖啡/奶香 → #8c6648 咖啡棕（极淡）
  - 茶/梅酒 → #7f9f69 抹茶绿（极淡）
  - 果酒/甜口 → #e0945a 珊瑚橙（极淡）
  - 百利甜/甜品 → #9b6a48 奶油棕（极淡）

### 6.4 留白与呼吸感

```
这是之前 v1/v2 最容易出错的地方：

✅ 必须留白的区域：
  - 图片边缘 10% 安全区（不放重要主体）

  - 画面填充率不超过 65%（35%+ 应该是"空"或"极淡底色"）
  - 主体周围要有"呼吸圈"（一圈淡淡的环境色晕染）
  - 不要把画面塞满，宁可不画

✅ 特别注意：
  - 小程序屏幕只有 375-414pt 宽，信息密度要低
  - 图片里不要画文字（文字由前端渲染，画了就固定死了）
  - 不要画按钮/边框/箭头（这些全是 CSS 实现）
```
